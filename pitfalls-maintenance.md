# AnythingLLM 踩坑记录维护

主文件：`anythingllm-skill-pitfalls.md`

## 何时维护

- 如果这次调用不顺利，但最终定位出原因，要补一条踩坑记录
- 如果先读了踩坑表，且其中某条记录**直接帮助定位、排除或修正问题**，就更新该条的查询次数

## 记录格式

每条记录固定为：**查询次数 / 现象 / 原因 / 正确做法**

必填字段：

- `查询次数：<number>`
- `现象：...`
- `原因：...`
- `正确做法：...`

## 查询次数规则

- 新增记录时：`查询次数：0`
- 某条记录在后续排查中直接帮助定位、排除或修正问题时：该条 `查询次数 +1`
- 排序暂不强制调整；默认先关注查询次数更高的项

## 维护要求

- 记录要短，只保留可复用信息
- 不写真实密钥、绝对路径、用户目录
- 不把长日志原样贴进踩坑表
- 如果是 CLI 误用，优先写成“错误写法 vs 正确写法”
